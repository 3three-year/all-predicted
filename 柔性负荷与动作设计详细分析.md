## 二、柔性负荷基类设计（FlexibleLoad）

### 2.1 基类架构

所有柔性负荷都继承自`FlexibleLoad`基类，实现了统一的功率调节机制：

```python
class FlexibleLoad:
    def __init__(self, max_power, min_power, response_time, ramp_rate):
        self.max_power = max_power      # 最大可调功率（kW）
        self.min_power = min_power      # 最小可调功率（kW），支持负值
        self.response_time = response_time  # 响应时间（分钟）
        self.ramp_rate = ramp_rate      # 调节速率（kW/分钟）
        self.current_power = 0.0        # 当前功率（kW）
        self.target_power = 0.0         # 目标功率（kW）
        self.adjust_progress = 0.0      # 调节进度（0~1）
```

### 2.2 核心特性

1. **双向功率调节**：支持正负功率值，实现真正的削峰（负功率）和填谷（正功率）
2. **渐进式响应**：通过`response_time`和`ramp_rate`实现平滑的功率变化，避免突变
3. **目标跟踪机制**：通过`adjust_progress`跟踪功率调节进度，确保平滑过渡

## 三、空调柔性负荷（AirConditioningLoad）详细设计

### 3.1 类定义与初始化

```python
class AirConditioningLoad(FlexibleLoad):
    def __init__(self, max_power, min_power, response_time, ramp_rate, comfort_temp):
        super().__init__(max_power, min_power, response_time, ramp_rate)
        self.comfort_temp = comfort_temp  # 用户舒适温度（℃），默认26.0℃
        self.current_temp = 25.0          # 当前室内温度（℃）
```

**初始化参数**（在环境中的实际设置）：
- `max_power = 15.0 kW`：最大制冷功率
- `min_power = -10.0 kW`：最小功率（负值表示可降低负荷，实现削峰）
- `response_time = 3 分钟`：响应时间短，适合快速调节
- `ramp_rate = 2.0 kW/分钟`：调节速率快
- `comfort_temp = 26.0 ℃`：用户舒适温度

### 3.2 温度动态模型

空调负荷的核心创新在于**温度-功率耦合模型**：

```python
def update_temperature(self, delta_time, outdoor_temp):
    """模拟室内温度变化（简化热力学模型）"""
    # 温度变化率与功率相关（假设空调功率与制冷量成正比）
    temp_change_rate = 0.1 * (self.current_power / self.max_power) - 0.05 * (self.current_temp - outdoor_temp)
    self.current_temp += temp_change_rate * delta_time
    self.current_temp = np.clip(self.current_temp, 16.0, 30.0)  # 限制温度范围
    return self.current_temp
```

**模型特点**：
- **功率-温度耦合**：空调功率直接影响室内温度变化率
- **热力学平衡**：考虑室内外温差对温度变化的影响
- **温度约束**：限制在16-30℃范围内，保证用户基本舒适度

### 3.3 用户满意度计算

空调负荷通过**热舒适度**影响用户满意度：

```python
def _calculate_user_satisfaction(self):
    """计算用户综合满意度"""
    # 热舒适度（基于PMV模型简化）
    temp_diff = abs(self.ac_load.current_temp - self.ac_load.comfort_temp)
    thermal_comfort = np.exp(-0.1 * temp_diff)  # 指数函数，温度差越小舒适度越高
    
    # 根据时段动态调整权重
    if self._is_peak_hour():
        thermal_weight = self.thermal_comfort_weight * self.peak_hour_weight  # 高峰时段降低权重
    elif self._is_valley_hour():
        thermal_weight = self.thermal_comfort_weight * self.valley_hour_weight  # 低谷时段提升权重
    
    satisfaction = thermal_weight * thermal_comfort + discharge_weight * self.ev_load.soc
    return satisfaction
```

**创新点**：
- **时段自适应权重**：高峰时段降低热舒适度权重，优先保障供电；低谷时段提升权重，兼顾用户体验
- **指数衰减模型**：温度偏离舒适温度时，舒适度指数衰减，符合人体感知特性

### 3.4 动作2：空调削峰填谷策略

当强化学习智能体选择**动作2（调整空调设定温度）**时，系统执行以下智能策略：

#### 3.4.1 光伏充足时的填谷策略

```python
if pv_output > 3.0:  # 光伏充足时，积极填谷
    if current_load > 1.2:  # 负荷较高时，适度增加空调功率消纳光伏
        pv_factor = min(0.8, pv_output / 15.0)
        target_power = self.ac_load.max_power * (0.5 + 0.6 * pv_factor)  # 平衡的填谷能力
    else:  # 负荷较低时，适度增加空调功率填谷
        pv_factor = min(0.9, pv_output / 15.0)
        target_power = self.ac_load.max_power * (0.6 + 0.7 * pv_factor)  # 平衡的填谷能力
```

**策略逻辑**：
- **光伏消纳优先**：当光伏出力充足时，增加空调功率消耗多余光伏，提高消纳率
- **动态功率调整**：根据光伏出力大小动态调整空调功率，实现最优消纳

#### 3.4.2 光伏不足时的削峰策略

系统根据光伏出力水平采用**分级削峰策略**：

```python
if pv_output < 1.0:  # 光伏很低时（低于1kW）- 应该积极削峰
    if current_load > 0.5:
        reduction_factor = min(0.7, (current_load - 0.5) / 1.5)  # 提高削减强度
        max_reduction = min(self.load_demand * 0.4, self.ac_load.max_power * 0.6)
        target_power = -max_reduction * reduction_factor  # 负值削峰
elif pv_output < 2.0:  # 光伏较低时
    # 类似的削峰逻辑，但削减强度略低
elif time_step >= 60:  # 后面时间段 - 光伏低谷时应该积极削峰
    # 针对60-80时间步区间进行积极削峰
    if 60 <= time_step <= 80:
        reduction_factor = min(0.8, (current_load - 0.3) / 1.0)
        max_reduction = min(self.load_demand * 0.5, self.ac_load.max_power * 0.7)
        target_power = -max_reduction * reduction_factor
```

**创新点**：
1. **分级削峰**：根据光伏出力水平（<1kW, <2kW, 其他）采用不同削峰强度
2. **时段感知**：针对光伏低谷时段（60-80时间步）采用更积极的削峰策略
3. **负功率削峰**：通过设置负功率值，直接降低系统总负荷，实现真正的削峰效果

#### 3.4.3 平滑控制机制

为防止功率突变，系统采用**指数移动平均**和平滑约束：

```python
if hasattr(self, 'prev_ac_power'):
    # 使用更强的指数移动平均来平滑功率变化
    alpha = 0.2  # 更强的平滑因子，进一步减少波动
    target_power = alpha * target_power + (1 - alpha) * self.prev_ac_power
    
    # 添加额外的平滑约束：限制单步变化幅度
    max_change = self.ac_load.max_power * 0.2  # 单步最大变化为最大功率的20%
    power_diff = target_power - self.prev_ac_power
    if abs(power_diff) > max_change:
        target_power = self.prev_ac_power + np.sign(power_diff) * max_change
```

**优势**：
- **减少功率波动**：避免空调功率频繁大幅变化，保护设备
- **提升用户体验**：平滑的温度变化更符合用户期望
- **系统稳定性**：减少对电网的冲击

## 四、电动汽车（EV）柔性负荷详细设计

    def __init__(self, max_power, min_power, response_time, ramp_rate, soc, capacity):
        super().__init__(max_power, min_power, response_time, ramp_rate)

        self.capacity = capacity      # 电动汽车电池容量（kWh），默认200.0 kWh

**初始化参数**（在环境中的实际设置）：
- `max_power = 20.0 kW`：最大充电功率
- `min_power = -15.0 kW`：最小功率（负值表示V2G放电）
- `response_time = 8 分钟`：响应时间较长，符合EV充电特性
- `ramp_rate = 1.5 kW/分钟`：调节速率适中
- `soc = 0.6`：初始SOC为60%
- `capacity = 200.0 kWh`：电池容量200 kWh

### 4.2 SOC动态更新模型

EV负荷的核心是**SOC（State of Charge）管理**：

```python
def update_soc(self, delta_time):
    """更新SOC（充电或放电）"""
    if self.current_power > 0:  # 充电
        self.soc += (self.current_power * delta_time) / self.capacity
    elif self.current_power < 0:  # 放电（V2G）
        self.soc -= (abs(self.current_power) * delta_time) / self.capacity
    self.soc = np.clip(self.soc, 0.0, 1.0)
```

**特点**：
- **双向能量流**：支持充电（正功率）和V2G放电（负功率）
- **能量守恒**：SOC变化与功率和时间成正比，符合物理规律
- **安全约束**：SOC限制在0-1范围内

### 4.4 动作3：EV智能调控策略

当强化学习智能体选择**动作3（调整电动汽车充放电功率）**时，系统执行以下智能策略：

**创新点**：
- **进度跟踪**：实时跟踪实际充电量与期望充电量的比例

- **约束满足**：保证全天充电总量与预测值一致，符合实际应用需求

3. **削峰能力**：光伏不足时减少充电功率或启用V2G放电，实现削峰

### 5.2 动作设计创新点

#### 5.2.1 离散动作空间

采用**离散动作空间**而非连续动作空间，具有以下优势：
- **降低学习难度**：离散动作空间更容易学习，收敛更快
- **实际应用友好**：实际系统中，控制指令通常是离散的（如"增加功率"、"减少功率"）
- **可解释性强**：每个动作有明确的物理意义，便于理解和调试

#### 5.2.2 动作分离设计

将**储能动作**和**柔性负荷动作**分离，实现：
- **职责清晰**：储能负责能量存储，柔性负荷负责负荷调节
- **并行控制**：可以同时控制多个系统，提高灵活性
- **策略组合**：智能体可以学习不同动作的组合策略

#### 5.2.3 智能动作执行

每个动作不是简单的"开/关"，而是**智能策略执行**：
- **动作2（空调）**：根据光伏出力、负荷水平、时段等因素智能计算目标功率
- **动作3（EV）**：基于真实EV数据，考虑SOC、充电进度、光伏情况等因素智能调控

## 六、核心创新点总结

### 6.1 空调柔性负荷创新点

1. **双向功率调节**：支持正负功率值，实现真正的削峰填谷
2. **温度-功率耦合模型**：建立温度动态模型，考虑用户热舒适度
3. **时段自适应权重**：根据峰谷时段动态调整热舒适度权重
4. **分级削峰策略**：根据光伏出力和时段采用不同削峰强度
5. **平滑控制机制**：通过指数移动平均和平滑约束减少功率波动

## 八、奖励函数中的柔性负荷考虑

### 8.1 用户满意度奖励

柔性负荷通过**用户满意度**直接影响奖励函数：

```python
user_satisfaction = self._calculate_user_satisfaction()
# 用户满意度 = 热舒适度权重 * 热舒适度 + 放电意愿权重 * EV的SOC

user_reward = user_satisfaction * (0.7 + 0.2 * difficulty)
```

**机制**：
- **热舒适度**：空调温度偏离舒适温度时，用户满意度降低，奖励减少
- **EV SOC**：EV的SOC影响用户满意度，SOC过低时满意度降低
- **时段权重**：高峰时段降低热舒适度权重，低谷时段提升权重

### 8.2 削峰填谷奖励

柔性负荷的削峰填谷效果通过**峰谷差奖励**体现：

```python
# 削峰填谷综合奖励
peak_valley_reward = (
    peak_penalty + peak_bonus + valley_bonus + valley_penalty +
    pv_load_matching_reward + flexible_negative_reward + pv_low_reduction_reward
)
```

**奖励项说明**：
- **peak_penalty**：高峰时段负荷过高时惩罚
- **valley_bonus**：低谷时段增加负荷时奖励（填谷）
- **flexible_negative_reward**：柔性负荷负功率削峰时的奖励
- **pv_low_reduction_reward**：光伏不足时减少负荷的奖励

### 8.3 光伏消纳奖励

柔性负荷通过**增加负荷**帮助消纳光伏，获得奖励：

```python
# 动态光伏消纳奖励
dynamic_utilization_bonus = 0.15  # 基础奖励系数
pv_reward = dynamic_utilization_bonus * pv_used  # 消纳的光伏越多，奖励越高
```

**机制**：
- **填谷消纳**：光伏充足时，空调和EV增加功率消耗光伏，获得消纳奖励
- **动态奖励系数**：根据电价动态调整，谷电价时奖励更高

### 8.4 负荷平滑度奖励

柔性负荷的**平滑控制**通过平滑度奖励得到强化：

```python
# T-DuelingDDQN专用平滑度奖励
if hasattr(self, 'prev_total_load'):
    load_change = abs(total_load - self.prev_total_load)
    smoothness_reward = max(0, 1.0 - load_change / 2.0) * 0.5  # 变化越小奖励越高
    peak_valley_reward += smoothness_reward
```

**意义**：
- **鼓励平滑**：负荷变化越小，奖励越高
- **系统稳定**：减少功率突变，提高系统稳定性

## 五补. 电动汽车调控策略论文稿

为了便于写作与排版，本节整理成论文用稿的形式，包括流程描述、公式与参数说明。

### 5补.1 调控流程综述

为保证电动汽车充电总量满足预测需求并兼顾光伏消纳与电网削峰，设计了基于进度偏差的分层调控策略。策略以真实预测曲线为基准，实时计算“期望进度”与“实际进度”的差异，并结合光伏出力、系统负荷与电池荷电状态（SOC）进行功率补偿，使日内充放电能量守恒，同时提升光伏消纳率与峰谷调节能力。

### 5补.2 数学描述

设一天被划分为 $T = 96$ 个 15 分钟步长，$E^{\mathrm{ref}}$ 为参考总充电能量，$E^{\mathrm{act}}_t$ 为截至时刻 $t$ 的累计充电量。

1. **进度守恒约束**

\[
\eta^{\mathrm{exp}}_t = \frac{t}{T},\quad \eta^{\mathrm{act}}_t = \frac{E^{\mathrm{act}}_t}{E^{\mathrm{ref}}},\quad \delta_t = \eta^{\mathrm{act}}_t - \eta^{\mathrm{exp}}_t.
\]

补偿功率按照进度偏差分段调整：

\[
P^{\mathrm{soc}}_t =
\begin{cases}
\kappa_{\mathrm{fast}}\, P^{\max}, & \delta_t < -\theta_{\mathrm{lag}} \text{ 且 } \text{SOC}_t < \text{SOC}^{\max},\\[4pt]
\kappa_{\mathrm{slow}}\, P^{\mathrm{ref}}_t, & \delta_t > \theta_{\mathrm{lead}},\\[4pt]
P^{\mathrm{ref}}_t, & \text{otherwise}.
\end{cases}
\]

- $P^{\mathrm{ref}}_t$：参考功率（22年真实EV数据）。
- $P^{\max}$：充电功率上限（约 20 kW）。
- $\kappa_{\mathrm{fast}}$、$\kappa_{\mathrm{slow}}$：调节系数（实验中取 0.8 与 0.5）。
- $\theta_{\mathrm{lag}}$、$\theta_{\mathrm{lead}}$：进度偏差阈值（取 0.2）。

2. **光伏与负荷响应**

设 $\theta_{\mathrm{PV}}$ 为光伏触发阈值，$\theta_{\mathrm{ld}}$ 为负荷阈值，$(x)^+ = \max(x,0)$：

\[
\Delta P^{\mathrm{pv}}_t =
\begin{cases}
\beta_{\mathrm{pv}}\, (\text{PV}_t - \theta_{\mathrm{PV}})^+, & \text{SOC}_t < \text{SOC}^{\mathrm{up}},\\[4pt]
0, & \text{PV}_t > \theta_{\mathrm{PV}} \text{ 且 } \text{SOC}_t \geq \text{SOC}^{\mathrm{up}},\\[4pt]
-\beta_{\mathrm{ld}}\, (L_t - \theta_{\mathrm{ld}})^+, & \text{SOC}_t > \text{SOC}^{\mathrm{down}},\\[4pt]
0, & \text{otherwise}.
\end{cases}
\]

- $\beta_{\mathrm{pv}}$, $\beta_{\mathrm{ld}}$：调节系数（取 0.1–0.3）。
- $\text{PV}_t$：光伏出力（kW）；$L_t$：归一化系统负荷。
- $\text{SOC}^{\mathrm{up}}$、$\text{SOC}^{\mathrm{down}}$：上/下限阈值（取 0.8、0.3）。
- $\theta_{\mathrm{PV}}$、$\theta_{\mathrm{ld}}$：通过光伏出力与负荷分位数确定（实验中分别约 3 kW、0.6）。

3. **目标功率与平滑控制**

\[
P^{\mathrm{target}}_t = \mathrm{clip}\bigl( P^{\mathrm{soc}}_t + \Delta P^{\mathrm{pv}}_t,\ P^{\min},\ P^{\max} \bigr).
\]

\[
P^{\mathrm{cmd}}_t = \alpha P^{\mathrm{target}}_t + (1-\alpha) P^{\mathrm{cmd}}_{t-1}, \quad |P^{\mathrm{cmd}}_t - P^{\mathrm{cmd}}_{t-1}| \leq \Delta P^{\max}.
\]

- $P^{\min}$：放电功率下限（约 -15 kW）。
- $\alpha$：平滑系数（Transformer 算法取 0.2，其余取 0.25–0.3）。
- $\Delta P^{\max}$：单步最大功率变化（取最大功率的 20%）。

### 5补.3 参数取值说明

| 参数 | 物理含义 | 取值依据与示例 |
| --- | --- | --- |
| $P^{\max}$ | EV 充电功率上限 | 根据桩额定功率，实验中取 20 kW |
| $P^{\min}$ | V2G 放电功率下限 | 设备支持度量，取 -15 kW |
| $\theta_{\mathrm{lag}}$, $\theta_{\mathrm{lead}}$ | 进度偏差阈值 | 经验设为 0.2，对应允许 ±20% 偏差 |
| $\kappa_{\mathrm{fast}}$, $\kappa_{\mathrm{slow}}$ | 补偿系数 | 0.8/0.5，分别对应加速/减缓充电 |
| $\theta_{\mathrm{PV}}$ | 光伏触发阈值 | 由光伏出力分位数估计，实测约 3 kW |
| $\theta_{\mathrm{ld}}$ | 负荷触发阈值 | 峰段负荷分位数，实测约 0.6 |
| $\beta_{\mathrm{pv}}$, $\beta_{\mathrm{ld}}$ | 光伏/负荷调节率 | 选取 0.1–0.3 以平衡响应速度 |
| $\text{SOC}^{\mathrm{up}}$, $\text{SOC}^{\mathrm{down}}$ | SOC 上下限 | 满足用户需求，取 0.8 和 0.3 |
| $\alpha$ | 平滑系数 | 与算法相关，0.2–0.3 |
| $\Delta P^{\max}$ | 单步变化限制 | 取 $0.2 P^{\max}$ |
