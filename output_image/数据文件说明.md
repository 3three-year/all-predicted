# 输出数据文件说明

## 📊 生成的数据文件

运行 `train_simplified.py` 后，会在 `output_image/` 目录生成以下数据文件：

### 1. 逐时数据文件
**文件名格式**：`典型日X_MM-DD_逐时数据.xlsx`

**内容**：包含96个时间点（每15分钟一个）的详细数据

**列说明**：
| 列名 | 说明 | 单位 |
|------|------|------|
| 时间点 | 0-95（96个点） | - |
| 时刻 | HH:MM格式 | - |
| 小时 | 0.00-23.75 | 小时 |
| {算法}_总负荷(kW) | 总负荷功率 | kW |
| {算法}_光伏出力(kW) | 光伏发电功率 | kW |
| {算法}_净负荷(kW) | 需要从电网购电的功率 | kW |
| {算法}_储能SOC | 储能电池荷电状态 | 0-1 |
| {算法}_电价(元/kWh) | 当前电价 | 元/kWh |
| {算法}_光伏消纳率 | 光伏自用比例 | 0-1 |
| {算法}_EV负荷(kW) | EV充放电功率（正=充电，负=放电） | kW |

**算法列表**：Baseline, DDQN, T-DDQN, Ablation, PPO

**用途**：
- 绘制详细的时序曲线
- 分析特定时段的运行状态
- 验证算法的调度策略

### 2. 汇总指标文件
**文件名格式**：`典型日X_MM-DD_汇总指标.xlsx`

**内容**：每个算法的关键性能指标

**指标说明**：

#### 经济性指标
| 指标 | 说明 | 单位 |
|------|------|------|
| 总成本(元) | 一天的总购电成本 | 元 |
| 成本降低率(%) | 相对Baseline的成本降低百分比 | % |

**计算公式**：
```
总成本 = Σ(净负荷 × 电价 × 0.25小时)
成本降低率 = (Baseline成本 - 算法成本) / Baseline成本 × 100%
```

#### 光伏消纳指标
| 指标 | 说明 | 单位 |
|------|------|------|
| 平均光伏消纳率(%) | 光伏自用比例的平均值 | % |
| 光伏消纳提升(%) | 相对Baseline的提升百分点 | 百分点 |
| 总光伏发电量(kWh) | 一天的总光伏发电量 | kWh |

**计算公式**：
```
光伏消纳率 = min(光伏出力, 总负荷) / 光伏出力
平均光伏消纳率 = Σ(光伏消纳率) / 96
```

#### 削峰填谷指标
| 指标 | 说明 | 单位 |
|------|------|------|
| 峰值负荷(kW) | 一天中的最大负荷 | kW |
| 谷值负荷(kW) | 一天中的最小负荷 | kW |
| 峰谷差(kW) | 峰值负荷 - 谷值负荷 | kW |
| 峰谷差降低率(%) | 相对Baseline的峰谷差降低百分比 | % |

**计算公式**：
```
峰谷差 = max(负荷) - min(负荷)
峰谷差降低率 = (Baseline峰谷差 - 算法峰谷差) / Baseline峰谷差 × 100%
```

#### EV调度指标
| 指标 | 说明 | 单位 |
|------|------|------|
| EV总充电量(kWh) | 一天的总充电电量 | kWh |
| EV总放电量(kWh) | 一天的总放电电量（V2G） | kWh |
| EV最大充电功率(kW) | 最大充电功率 | kW |
| EV最大放电功率(kW) | 最大放电功率 | kW |

**计算公式**：
```
EV总充电量 = Σ(max(0, EV负荷)) × 0.25小时
EV总放电量 = Σ(abs(min(0, EV负荷))) × 0.25小时
```

#### 储能指标
| 指标 | 说明 | 单位 |
|------|------|------|
| 储能SOC范围 | SOC的最大值 - 最小值 | - |
| 储能平均SOC | SOC的平均值 | - |

## 🔍 数据验证方法

### 验证成本计算
```python
import pandas as pd

# 读取逐时数据
df = pd.read_excel('典型日1_09-16_逐时数据.xlsx')

# 手动计算DDQN的成本
total_cost = 0
for i in range(96):
    net_load = df.loc[i, 'DDQN_净负荷(kW)']
    price = df.loc[i, 'DDQN_电价(元/kWh)']
    total_cost += net_load * price * 0.25

print(f"手动计算的成本: {total_cost:.2f} 元")

# 读取汇总指标
df_summary = pd.read_excel('典型日1_09-16_汇总指标.xlsx')
ddqn_cost = df_summary[df_summary['算法'] == 'DDQN']['总成本(元)'].values[0]
print(f"汇总文件中的成本: {ddqn_cost:.2f} 元")

# 两者应该一致
```

### 验证光伏消纳率
```python
# 读取逐时数据
df = pd.read_excel('典型日1_09-16_逐时数据.xlsx')

# 计算平均光伏消纳率
avg_pv_util = df['DDQN_光伏消纳率'].mean()

```

## 📝 使用建议

### 1. 论文写作
- 使用**汇总指标文件**中的数据填写表格
- 引用具体数值时保留2位小数
- 对比不同算法时使用改善率指标

- 使用**逐时数据文件**绘制自定义图表
- 分析特定时段（如峰时、谷时）的策略差异
- 研究算法的动态响应特性

### 3. 结果验证
- 对比逐时数据和汇总指标，确保一致性
- 检查EV充放电是否满足能量守恒
- 验证储能SOC是否在合理范围内（0.1-0.9）

### 场景2：分析峰时策略
```python
# 读取逐时数据
df = pd.read_excel('典型日1_09-16_逐时数据.xlsx')

# 提取峰时数据（8-11点，18-22点）
peak_hours = df[(df['小时'] >= 8) & (df['小时'] < 11) | 
                (df['小时'] >= 18) & (df['小时'] < 22)]

# 分析各算法在峰时的EV策略
print("峰时EV平均功率:")
for algo in ['Baseline', 'DDQN', 'T-DDQN', 'PPO']:
    avg_ev = peak_hours[f'{algo}_EV负荷(kW)'].mean()
    print(f"  {algo}: {avg_ev:.2f} kW")
```

## 📞 常见问题

**Q1: 为什么有些算法的EV负荷列为空？**
A: 如果算法没有返回EV负荷数据，该列会为空。Baseline通常不调控EV，所以使用真实EV负荷。

**Q2: 光伏消纳率为什么有时是0？**
A: 当光伏出力小于5kW时，为避免小数值导致的高消纳率误导，设置为0。

**Q3: 净负荷为什么不会是负值？**
A: 净负荷 = max(0, 总负荷 - 光伏)，只计算需要从电网购电的部分。

**Q4: 如何引用这些数据？**
A: 在论文中可以这样写："如表X所示，T-DDQN算法在典型日1（09-16）实现了415.8元的运行成本，相比Baseline降低25.4%。"
