## 二、三大核心奖励组件详解

### 2.1 经济性奖励 $r_{cost}$

**对应图片公式：**
$$r_{cost} = -(c_{energy} \cdot L_{负荷} + c_{carbon} \cdot L_{负荷})$$

**代码实现（第1246、1970-1971行）：**
```python
# 能源成本
energy_cost = self._get_electricity_price() * net_load * 0.25  # 15分钟=0.25小时

# 碳成本
carbon_cost = self.carbon_price * net_load  # 净负荷的碳排放成本

# 总成本惩罚
cost_penalty = -(energy_cost + carbon_cost) * 2.0

# 经济性奖励（第1983行）
economic_reward = -energy_cost * (0.5 + 0.3 * difficulty)
```

**详细说明：**

#### （1）$c_{energy}$：电价（元/kWh）

**获取方式：** 通过 `_get_electricity_price()` 函数动态计算（第873-917行）

**基础分时电价：**
```python
# 峰时段（8-11点，18-22点）
if 8 <= self.current_hour < 11 or 18 <= self.current_hour < 22:
    base_price = self.peak_price  # 0.55元/kWh

# 谷时段（23-7点）
elif 23 <= self.current_hour or self.current_hour < 7:
    base_price = self.valley_price  # 0.30元/kWh

# 平时段（7-8点，11-18点，22-23点）
else:
    base_price = (self.peak_price + self.valley_price) / 2  # 0.425元/kWh
```

**动态调整因子：**

电价会根据多个因素动态调整：

1. **季节性因子**：
   ```python
   if month in [6, 7, 8, 12, 1, 2]:  # 夏季和冬季
       seasonal_factor = 1.15  # 上浮15%
   else:
       seasonal_factor = 0.95  # 下浮5%
   ```

2. **日期类型因子**：
   ```python
   if weekday >= 5:  # 周末
       day_factor = 0.9  # 下浮10%
   else:
       day_factor = 1.05  # 上浮5%
   ```

3. **负荷因子**：
   ```python
   load_factor = np.clip(self.load_demand / self.max_load.mean(), 0.8, 1.2)
   # 负荷越高，电价越高（0.8-1.2倍）
   ```

4. **光伏因子**：
   ```python
   pv_factor = 1.0 - (self._get_pv_utilization_ratio() / 2.0)
   # 光伏消纳率越高，电价越低
   ```

5. **SOC因子**：
   ```python
   soc_factor = 1.1 - (self.battery_soc * 0.5)
   # SOC越高，电价越低（鼓励放电）
   ```

**最终电价计算：**
```python
dynamic_price = base_price * seasonal_factor * day_factor * load_factor * pv_factor * soc_factor
# 限制在合理范围内
final_price = np.clip(dynamic_price, self.valley_price * 0.7, self.peak_price * 1.3)
# 即：0.21元/kWh - 0.715元/kWh
```

**$c_{energy}$ 的取值范围：**
- **最低**：0.21元/kWh（谷时 × 0.7）
- **基础谷时**：0.30元/kWh
- **基础平时**：0.425元/kWh
- **基础峰时**：0.55元/kWh
- **最高**：0.715元/kWh（峰时 × 1.3）

#### （2）$c_{carbon}$：碳成本系数（元/kWh）

**获取方式：** 在环境初始化时设置（第399行）

```python
self.carbon_price = 0.1  # 碳成本系数（元/kWh）
```

**$c_{carbon}$ 的固定值：**
- **碳成本系数**：0.1元/kWh

**物理意义：**
- 假设从电网购买的电力主要来自火电等高碳能源
- 每消耗1 kWh电力，产生一定量的碳排放
- 碳成本系数0.1元/kWh表示碳排放的经济成本

**碳成本计算（第1970行）：**
```python
carbon_cost = self.carbon_price * net_load
# 即：0.1元/kWh × 净负荷(kW) × 0.25小时
```

#### （3）$L_{负荷}$：净负荷（kW）

**计算方式：**
```python
net_load = max(0, total_load - pv_output)
```

**物理意义：**
- 当总负荷大于光伏出力时，需要从电网购电
- 净负荷 = 总负荷 - 光伏出力
- 净负荷越大，需要从电网购买的电力越多

#### （4）总成本计算

**完整公式：**
```python
# 能源成本（元）
energy_cost = c_energy × net_load × 0.25小时

# 碳成本（元）
carbon_cost = c_carbon × net_load × 0.25小时

# 总成本（元）
total_cost = energy_cost + carbon_cost
            = (c_energy + c_carbon) × net_load × 0.25
            = (0.21~0.715 + 0.1) × net_load × 0.25
            = (0.31~0.815) × net_load × 0.25
```

**成本奖励：**
```python
# 方式1：直接成本惩罚（第1971行）
cost_penalty = -(energy_cost + carbon_cost) * 2.0

# 方式2：经济性奖励（第1983行，实际使用）
economic_reward = -energy_cost * (0.5 + 0.3 * difficulty)
```

**注意：** 在实际代码中，主要使用的是 `economic_reward`，而 `carbon_cost` 在某些版本中被注释掉或未直接使用。

**与图片公式的对应关系：**
- 图片中的 $c_{energy}$：对应代码中的 `_get_electricity_price()`，动态范围0.21-0.715元/kWh
- 图片中的 $c_{carbon}$：对应代码中的 `self.carbon_price = 0.1`元/kWh
- 图片中的 $L_{负荷}$：对应代码中的 `net_load × 0.25`（kWh）

### 2.2 环保性奖励 $r_{pv}$

**对应图片公式(46)：**
$$r_{pv} = \lambda_{util} \cdot P_{used} - \lambda_{curt} \cdot P_{curtailed}$$

**代码实现（第1275-1279行）：**
```python
pv_reward = (
    dynamic_utilization_bonus * pv_used +      # λ_util * P_used
    self.curtailment_penalty * final_curtailment  # λ_curt * P_curtailed
)
```

**详细说明：**

1. **光伏消纳奖励**：
   ```python
   pv_used = min(pv_output, total_load)  # 实际消纳的光伏电量
   dynamic_utilization_bonus = 基础奖励系数（根据光伏出力动态调整）
   ```
   - 消纳的光伏越多，奖励越高
   - 鼓励算法在光伏高峰期增加负荷

2. **弃光惩罚**：
   ```python
   final_curtailment = max(0, pv_output - total_load)  # 弃光量
   self.curtailment_penalty = -0.2  # 弃光惩罚系数（负值）
   ```
   - 弃光越多，惩罚越大
   - 鼓励算法最大化光伏消纳

3. **环境奖励调整（第1984行）**：
   ```python
   environmental_reward = pv_reward * (0.8 - 0.3 * difficulty)
   ```
   - 根据训练难度动态调整权重

**与图片公式的完全对应：**
- $\lambda_{util}$：对应 `dynamic_utilization_bonus`
- $P_{used}$：对应 `pv_used`
- $\lambda_{curt}$：对应 `self.curtailment_penalty`（-0.2）
- $P_{curtailed}$：对应 `final_curtailment`

### 2.3 用户体验奖励 $r_{user}$

**对应图片公式(46)：**
$$r_{user} = \alpha \cdot e^{-\kappa|\Delta T|} + \beta \cdot SOC_{EV}$$

**代码实现（第1985行）：**
```python
user_reward = user_satisfaction * (0.7 + 0.2 * difficulty)
```

其中 `user_satisfaction` 包含两部分：

**（1）温度舒适度 $\alpha \cdot e^{-\kappa|\Delta T|}$**

代码中通过空调负荷调控实现：
```python
# 温度偏差惩罚（隐含在user_satisfaction中）
temp_deviation = abs(current_temp - comfort_temp)
thermal_comfort = α * exp(-κ * temp_deviation)
```

**（2）EV SOC满意度 $\beta \cdot SOC_{EV}$**

代码中通过多个EV相关奖励实现：

1. **EV充电需求满足度奖励**（第2075-2110行）：
   ```python
   ev_energy_fulfillment_reward = 0
   if 0.9 <= fulfillment_ratio <= 1.1:
       ev_energy_fulfillment_reward = 2.0  # 充电量合理，给予奖励
   elif fulfillment_ratio < 0.9:
       shortage = 0.9 - fulfillment_ratio
       ev_energy_fulfillment_reward = -5.0 * shortage  # 充电不足，惩罚
   else:
       excess = fulfillment_ratio - 1.1
       ev_energy_fulfillment_reward = -2.0 * excess  # 过度充电，轻微惩罚
   ```

2. **EV充电进度对齐奖励**（第2110-2120行）：
   ```python
   ev_progress_reward = 0.0
   if getattr(self, 'reference_ev_energy', 0.0) > 1e-6:
       current_progress = self.actual_ev_energy_charged / self.reference_ev_energy
       expected_progress = self._expected_ev_completion(self.current_step)
       progress_gap = expected_progress - current_progress
       if progress_gap > 0:
           ev_progress_reward = -2.5 * progress_gap  # 进度落后，惩罚
       else:
           ev_progress_reward = min(1.0, -progress_gap * 1.5)  # 进度超前，奖励
   ```

**与图片公式的对应关系：**
- $\alpha$：温度舒适度权重（隐含在代码中）
- $e^{-\kappa|\Delta T|}$：温度偏差的指数衰减函数
- $\beta$：EV SOC权重（对应代码中的奖励系数2.0、-5.0等）
- $SOC_{EV}$：EV充电完成度（对应 `fulfillment_ratio`）

## 三、额外的奖励组件（代码扩展）

除了图片中的三大核心组件，代码还实现了以下额外奖励：

### 3.1 电池经济性奖励

```python
battery_economic_reward = 0
if battery_power_kw > 0.1:  # 充电
    charge_cost = battery_power_kw * 0.25 * current_price
    battery_economic_reward = -charge_cost * 1.2  # 惩罚充电，特别是高电价时
elif battery_power_kw < -0.1:  # 放电
    discharge_benefit = abs(battery_power_kw) * 0.25 * current_price
    battery_economic_reward = discharge_benefit  # 奖励放电，特别是高电价时
```

**设计目的：**
- 鼓励在谷时段充电（电价低）
- 鼓励在峰时段放电（电价高）
- 实现电价套利

### 3.2 削峰填谷奖励

```python
peak_valley_reward = (
    peak_penalty + peak_bonus + valley_bonus + valley_penalty +
    pv_load_matching_reward + flexible_negative_reward + pv_low_reduction_reward
)
```

**包含多个子组件：**

1. **峰时段惩罚/奖励**：
   - 峰时段负荷过高：惩罚
   - 峰时段负荷降低：奖励

2. **谷时段惩罚/奖励**：
   - 谷时段负荷过低：惩罚（浪费低电价）
   - 谷时段负荷增加：奖励（利用低电价）

3. **光伏-负荷匹配奖励**（第1880-1893行）：
   ```python
   if pv_output > 1.0:  # 光伏充足时
       load_pv_diff = abs(total_load - pv_output)
       pv_load_matching_reward = 0.5 * exp(-0.1 * load_pv_diff)  # 差值越小奖励越高
   else:  # 光伏很小时
       if total_load < 2.0:
           pv_load_matching_reward = 0.3 * (2.0 - total_load)  # 负荷越小奖励越高
   ```

4. **柔性负荷削峰奖励**（第1895-1908行）：
   ```python
   if total_flexible_power < 0:  # 削峰
       flexible_negative_reward = 2.0 * abs(total_flexible_power)
   if total_flexible_power > 0 and pv_output > 10.0:  # 填谷
       flexible_positive_reward = 1.5 * total_flexible_power
   if pv_output < 1.0 and total_flexible_power < 0:  # 光伏低谷削峰
       pv_low_reduction_reward = 2.5 * abs(total_flexible_power)
   ```

### 3.3 长期依赖奖励（Transformer特有）

```python
long_term_reward = 0
if len(self.state_history) >= 6:
    prev_state = np.array(self.state_history[-6])
    state_correlation = np.corrcoef(current_state, prev_state)[0, 1]
    long_term_reward = max(0, state_correlation) * 0.5 * difficulty
```

**设计目的：**
- 鼓励Transformer算法捕捉长时间跨度的依赖关系
- 奖励状态之间的相关性
- 提升时序建模能力

### 3.4 SOC平衡奖励

```python
soc_balance = 0
soc_penalty = 0
if self.battery_soc < 0.15:
    soc_penalty = -10.0 * (0.15 - self.battery_soc)  # SOC过低，重惩罚
elif self.battery_soc > 0.85:
    soc_penalty = -5.0 * (self.battery_soc - 0.85)  # SOC过高，轻惩罚
elif 0.4 <= self.battery_soc <= 0.6:
    soc_balance = 1.0  # SOC在理想范围，奖励
```

**设计目的：**
- 保持储能SOC在安全范围内（15%-85%）
- 鼓励SOC在理想范围内（40%-60%）
- 避免过充或过放

## 四、权重系数设计

### 4.1 算法特定权重

代码中为不同算法设置了不同的权重：

```python
current_weights = {
    'energy': 1.0,      # 经济性权重
    'pv': 0.8,          # 环保性权重
    'peak_valley': 0.6  # 削峰填谷权重
}
```

### 4.2 动态权重调整

根据训练难度动态调整权重：

```python
economic_reward = -energy_cost * (0.5 + 0.3 * difficulty)
environmental_reward = pv_reward * (0.8 - 0.3 * difficulty)
user_reward = user_satisfaction * (0.7 + 0.2 * difficulty)
```

**设计思想：**
- 训练初期（difficulty低）：更关注环保性和用户体验
- 训练后期（difficulty高）：更关注经济性

## 六、奖励函数设计的优点

1. **多目标平衡**：同时考虑经济性、环保性和用户体验
2. **动态权重**：根据训练难度自动调整各目标权重
3. **细粒度奖励**：包含多个子奖励组件，引导算法学习复杂策略
4. **算法特定**：为不同算法（DDQN、T-DDQN、PPO）设置不同权重
5. **长期依赖**：特别为Transformer设计了长期依赖奖励
6. **安全约束**：通过SOC惩罚确保储能安全运行

## 七、结论

项目中的奖励函数设计**与图片中的公式(45)(46)基本一致**，但在实现上更加详细和完善：

1. **核心框架一致**：$R = r_{cost} + r_{pv} + r_{user}$
2. **公式对应准确**：三大组件的计算方式与图片公式完全对应
3. **实现更丰富**：增加了电池经济性、削峰填谷、长期依赖等额外奖励
4. **权重更灵活**：采用动态权重和算法特定权重
5. **约束更完善**：增加了SOC安全约束、EV充电进度约束等

总体而言，代码实现是对图片公式的**忠实实现和合理扩展**，既保持了理论框架的一致性，又增加了实际应用所需的细节和约束。
