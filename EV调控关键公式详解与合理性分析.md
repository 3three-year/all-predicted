## 问题1：为什么要 × 0.25？（功率转电量）

### 🔍 问题重现

```python
# 代码中：
real_ev_load_day = [2.5, 2.3, 2.0, ..., 3.5]  # 96个时间点的功率值（kW）
self.reference_ev_energy = np.sum(np.maximum(real_ev_load_day, 0)) * 0.25

# 假设所有正值之和 = 340 kW
# reference_ev_energy = 340 * 0.25 = 85.0 kWh
```

### ✅ 详细解释

#### 物理意义：功率 × 时间 = 电量

**基础公式**：
```
电量（kWh）= 功率（kW）× 时间（小时）
Energy = Power × Time
```

**具体推导**：

**步骤1：理解数据结构**
```python
real_ev_load_day = [
    2.5,  # 时间步0：00:00-00:15，功率2.5 kW
    2.3,  # 时间步1：00:15-00:30，功率2.3 kW
    2.0,  # 时间步2：00:30-00:45，功率2.0 kW
    ...
    3.5   # 时间步95：23:45-24:00，功率3.5 kW
]
# 总共96个点，每个点代表15分钟的平均功率
```

**步骤2：计算每个时间步的电量**
```python
# 每个时间步的时长
Δt = 15分钟 = 15/60 小时 = 0.25 小时

# 第0个时间步的充电量
energy_0 = 2.5 kW × 0.25 h = 0.625 kWh

# 第1个时间步的充电量
energy_1 = 2.3 kW × 0.25 h = 0.575 kWh

# 第2个时间步的充电量
energy_2 = 2.0 kW × 0.25 h = 0.500 kWh

...
```

**步骤3：计算全天总充电量**
```python
# 方法1：分步计算
total_energy = energy_0 + energy_1 + energy_2 + ... + energy_95
             = (2.5 × 0.25) + (2.3 × 0.25) + (2.0 × 0.25) + ... + (3.5 × 0.25)
             = 0.25 × (2.5 + 2.3 + 2.0 + ... + 3.5)
             = 0.25 × 340
             = 85.0 kWh

# 方法2：向量化计算（代码中使用的）
total_energy = np.sum(real_ev_load_day) × 0.25
             = 340 × 0.25
             = 85.0 kWh
```

## 问题2：光伏不足时的 reduction_factor 公式

### 🔍 问题重现

```python
# 代码中：光伏不足且负荷较高时
if current_load > 0.6 and self.ev_load.soc > 0.3:
    reduction_factor = min(0.3, (current_load - 0.6) / 1.0)
    target_power = real_ev_power * (1 - reduction_factor)
```

### ✅ 详细解释

#### 设计思路：根据负荷高低决定削减幅度

**核心逻辑**：
```
负荷越高 → 削减幅度越大 → 削峰效果越好
负荷不高 → 削减幅度小   → 避免过度削减
```

#### 公式推导

**步骤1：计算 reduction_factor（削减系数）**

```python
reduction_factor = min(0.3, (current_load - 0.6) / 1.0)
```

**分解说明**：

```python
# 参数含义：
current_load = 当前基础负荷（kW）
0.6 = 削峰阈值（kW）- 超过此值才削峰
1.0 = 削峰敏感度（kW）- 控制削减速度
0.3 = 最大削减比例（30%）- 避免过度削减

# 计算过程：
超出部分 = current_load - 0.6
削减比例 = 超出部分 / 1.0
最终削减系数 = min(削减比例, 0.3)  # 限制在30%以内
```

**示例计算**：

| current_load | 超出部分 | 削减比例 | reduction_factor | 说明 |
|-------------|---------|---------|-----------------|------|
| 0.5 kW | -0.1 kW | -0.1 | **0（不削峰）** | 负荷很低，不需要削峰 |
| 0.6 kW | 0.0 kW | 0.0 | **0（不削峰）** | 刚好阈值，不削峰 |
| 0.8 kW | 0.2 kW | 0.2 | **0.2（削减20%）** | 略微超出，轻微削峰 |
| 1.0 kW | 0.4 kW | 0.4 | **0.3（削减30%）** | 超出较多，但限制在30% |
| 1.6 kW | 1.0 kW | 1.0 | **0.3（削减30%）** | 严重超出，最大削减30% |
| 3.2 kW | 2.6 kW | 2.6 | **0.3（削减30%）** | 极度超出，仍限制30% |

**步骤2：计算 target_power（目标功率）**

```python
target_power = real_ev_power * (1 - reduction_factor)
```

**示例计算**（假设 real_ev_power = 6.5 kW）：

| current_load | reduction_factor | target_power | 削减量 |
|-------------|-----------------|-------------|-------|
| 0.5 kW | 0 | 6.5 × (1-0) = **6.5 kW** | 0 kW（不削峰） |
| 0.8 kW | 0.2 | 6.5 × (1-0.2) = **5.2 kW** | -1.3 kW（削减20%） |
| 1.6 kW | 0.3 | 6.5 × (1-0.3) = **4.55 kW** | -1.95 kW（削减30%） |
| 3.2 kW | 0.3 | 6.5 × (1-0.3) = **4.55 kW** | -1.95 kW（削减30%） |

```
2. ✅ 0.6-1.6：线性增加削减比例（平滑过渡）
3. ✅ 负荷>1.6：限制在30%（避免过度削减）

```

### 🎯 设计原理

**1. 为什么选择 0.6 kW 作为阈值？**
```
- 台区基础负荷范围：通常0.5-10 kW
- 0.6 kW：相对较低的负荷水平
- 含义：只有负荷超过0.6 kW才考虑削峰
- 目的：避免在低负荷时段也削峰（没必要）
```

**2. 为什么用 1.0 作为敏感度系数？**
```
- 含义：负荷每增加1 kW，削减比例增加100%（即1.0）
- 效果：中等敏感度，既不过于激进，也不过于保守
- 调整空间：
  * 改为0.5 → 更激进（负荷每增加0.5kW，削减比例就增加100%）
  * 改为2.0 → 更保守（负荷每增加2kW，削减比例才增加100%）
```

**3. 为什么限制最大削减30%？**
```
原因1：保护用户体验
- 削减过多会影响用户充电需求
- 30%是可接受的调整范围

原因2：保持充电进度
- 如果削减过多（如50%），会导致总充电量不足
- 方案A会检测到进度落后，后续需要加速充电
- 30%的削减既能削峰，又不会严重影响进度

原因3：物理安全
- 频繁大幅度调整可能影响设备寿命
- 30%是相对温和的调整
```

| **平滑性** | ✅ 100分 | 线性过渡，避免突变 |

## 问题3：target_power 的计算逻辑

### 🔍 完整公式链

```python
# 完整的调控链条
real_ev_power = 6.5 kW  # 真实参考值

# 步骤1：根据负荷情况决定削减系数
reduction_factor = min(0.3, (current_load - 0.6) / 1.0)
# 假设 current_load = 3.2 kW
# reduction_factor = min(0.3, (3.2 - 0.6) / 1.0) = min(0.3, 2.6) = 0.3

# 步骤2：计算初步目标功率
target_power = real_ev_power * (1 - reduction_factor)
# target_power = 6.5 * (1 - 0.3) = 6.5 * 0.7 = 4.55 kW

# 步骤3：平滑控制（避免突变）
if hasattr(self, 'prev_ev_power'):
    alpha = 0.2  # 平滑系数
    target_power = alpha * target_power + (1 - alpha) * prev_ev_power
    # 假设 prev_ev_power = 5.0 kW
    # target_power = 0.2 * 4.55 + 0.8 * 5.0 = 0.91 + 4.0 = 4.91 kW

# 步骤4：单步变化限制
max_change = max_power * 0.2  # 最大变化 = 20 * 0.2 = 4.0 kW
power_diff = target_power - prev_ev_power  # 4.91 - 5.0 = -0.09 kW
if abs(power_diff) > max_change:
    # 不满足（0.09 < 4.0），不需要限制
    pass

# 步骤5：物理约束（确保在合理范围）
target_power = np.clip(target_power, min_power, max_power)
# target_power = np.clip(4.91, -15, 20) = 4.91 kW

# 最终：
target_power = 4.91 kW
```

### 📊 各步骤作用

| 步骤 | 作用 | 防止的问题 |
|------|------|----------|
| **步骤1** | 决定削减幅度 | 过度削减或削减不足 |
| **步骤2** | 应用削减 | 直接设定可能不合理 |
| **步骤3** | 平滑过渡 | 功率突变（设备冲击） |
| **步骤4** | 限制单步变化 | 极端情况下的剧烈调整 |
| **步骤5** | 物理约束 | 超出设备能力范围 |

```markdown
#### 3.4.1 充电需求量化

对于调度日d，基于历史EV充电功率曲线P_EV,ref^t（t=0,1,...,95），

#### 3.4.2 削峰策略

当系统负荷L_t超过削峰阈值L_th时，计算削减系数：

r_t = min(r_max, (L_t - L_th) / λ)    (公式2)

其中：
- r_max = 0.3 为最大削减比例，防止过度削减
- L_th = 0.6 kW 为削峰启动阈值
- λ = 1.0 kW 为削峰敏感度系数

P_EV^t = P_EV,ref^t × (1 - r_t)    (公式3)

ΔE_t = P_EV^t × Δt    (公式4)

E_actual = Σ_{t=0}^{95} max(P_EV^t, 0) × Δt    (公式5)

充电完成度：

η = E_actual / E_ref    (公式6)
```
