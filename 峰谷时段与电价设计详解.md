# 峰谷时段与电价设计详细解析

## 📚 **核心概念：峰谷时段指的是什么？**

### ⚡ **峰谷时段 = 负荷峰谷（用电需求），NOT 光伏峰谷**

**重要概念区分**：

| 概念 | 定义 | 特点 | 用途 |
|------|------|------|------|
| **负荷峰谷** | 用户用电需求的高低 | 与人类活动规律相关 | **决定电价** ✅ |
| **光伏峰谷** | 太阳能发电量的高低 | 与太阳辐射相关 | 决定发电量，不决定电价 |

### 🎯 **为什么峰谷时段基于负荷而非光伏？**

1. **电网调度需求**：
   - 电网需要在用电高峰时动员更多发电资源（成本高）
   - 用电低谷时发电资源闲置（成本低）
   - 通过价格信号引导用户错峰用电

2. **经济规律**：
   - **峰时（用电多）**：电网负担重 → **高电价**（限制需求）
   - **谷时（用电少）**：电网负担轻 → **低电价**（鼓励用电）

3. **与光伏的关系**：
   - 光伏在白天发电最多（11:00-14:00）
   - 但这不改变"负荷峰谷"的定义
   - 光伏只是"供给侧"的一个因素，不决定电价结构

## 📊 **当前峰谷时段与电价设计**

### 1. **时段划分（基于实际用电规律）**

```python
# rural_env.py 第993-999行

def _is_peak_hour(self):
    """峰时：用电高峰时段（8-11点，18-22点）"""
    return 8 <= self.current_hour < 11 or 18 <= self.current_hour < 22

def _is_valley_hour(self):
    """谷时：用电低谷时段（23-7点，即凌晨）"""
    return 23 <= self.current_hour or self.current_hour < 7
```

### 3. **电价计算公式**

```python
# rural_env.py 第848-859行

def _get_electricity_price(self):
    # 基础分时电价
    if 8 <= self.current_hour < 11 or 18 <= self.current_hour < 22:
        # 峰时
        base_price = self.peak_price  # 0.55元/kWh
    elif 23 <= self.current_hour or self.current_hour < 7:
        # 谷时
        base_price = self.valley_price  # 0.30元/kWh
    else:
        # 平时
        base_price = (self.peak_price + self.valley_price) / 2  # 0.425元/kWh
    
    # 动态调整（季节、日期、负荷、光伏、SOC等因素）
    dynamic_price = base_price * seasonal_factor * day_factor * load_factor * pv_factor * soc_factor
    
    return dynamic_price
```

## 🔋 **峰谷时段对储能SOC的影响**

### 1. **储能充放电策略逻辑**

```
谷时（00:00-06:59，23:00-23:59）：
  - 电价低 → ✅ 充电（从电网购电存储）
  - SOC ↑ 上升
  
峰时（08:00-10:59，18:00-21:59）：
  - 电价高 → ✅ 放电（释放存储的电能）
  - SOC ↓ 下降
  
平时（其他时段）：
  - 电价中等 → 根据光伏和负荷情况灵活决策
  - SOC 可能有小幅波动
```

## 💡 **关键洞察**

### 1. **峰谷电价的本质**

```
峰谷电价 = 基于"负荷峰谷"的价格信号
          ≠ 基于"光伏峰谷"

目的：引导用户错峰用电，平衡电网负荷
手段：价格差异（峰/谷 = 0.55/0.30 ≈ 1.8倍）
```

### 2. **光伏与峰谷的关系**

```
光伏峰值：11:00-14:00（发电最多）
负荷峰值：18:00-21:00（用电最多）
时间差：约6-8小时（错位！）

这种错位正是储能的价值所在：
  - 白天：光伏多 + 电价中等 → 弃光充电
  - 晚上：光伏无 + 电价高 → 放电削峰
```

### 3. **储能SOC变化的驱动因素**

```
主要驱动：电价（谷充峰放）
辅助驱动：弃光消纳（白天）
约束条件：SOC范围（15%-85%）

9月15日实例：
  - 谷时充电贡献：+25%（50%→75%）
  - 弃光充电贡献：+4%（75%→79%）
  - 峰时放电损失：-22%（79%→57%）
  - 净变化：+7%（略有盈余）
```

### 4. **EV负荷调控的双重目标**

```
目标1：总量守恒（方案A）
  - 保证全天充电总量满足需求
  - ∑(EV负荷调度后) = ∑(EV负荷调度前)

目标2：时间平移（削峰填谷）
  - 谷时多充（60%充电量）
  - 峰时少充或不充（0%充电量）
  - 平时正常（40%充电量）

9月15日实例：
  - 总需求：180kWh
  - 谷时充电：108kWh（60%，利用低价）
  - 平时充电：72kWh（40%，补充）
  - 峰时充电：0kWh（0%，完全避开）
  - 成本节省：约30元/天（相比均匀充电）
```

---

## 🎯 **总结：峰谷时段电价设计的核心逻辑**

1. **峰谷时段定义**：基于**负荷峰谷**（用电需求），而非光伏峰谷
   - 峰时 = 用电多的时段（8-11点，18-22点）
   - 谷时 = 用电少的时段（23-7点）

2. **电价信号作用**：引导用户错峰用电
   - 峰时高价 → 限制需求，鼓励放电
   - 谷时低价 → 鼓励用电，促进充电

3. **储能SOC响应**：跟随电价波动
   - 谷时充电（SOC↑）
   - 峰时放电（SOC↓）
   - 白天弃光充电（SOC小幅↑）

4. **EV负荷响应**：时间平移 + 总量守恒
   - 谷时多充（60%）
   - 峰时停充（0%）
   - 平时正常（40%）

5. **光伏的角色**：供给侧辅助，不决定电价结构
   - 白天发电，抵消部分负荷
   - 产生弃光时，可用于储能充电
   - 但不改变"峰谷时段"的基本定义

**9月15日是一个典型的"谷充峰放"案例，完美展示了峰谷电价引导下的储能和EV优化调度效果。** ✅
